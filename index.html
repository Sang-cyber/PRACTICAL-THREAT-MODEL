Trust Boundary 1
Internet → Frontend
Primary attack surface. All untrusted traffic enters here. Requires WAF, TLS termination, rate limiting, and input validation before any data proceeds inward.
CRITICAL
Trust Boundary 2
Backend → Databases
Privilege escalation risk. Application-tier code accesses sensitive data stores. SQL injection here results in full data compromise. Requires parameterized queries and least-privilege accounts.
CRITICAL
Trust Boundary 3
Corporate → Production
Highest systemic risk. A compromised corporate endpoint or AD account can pivot into production infrastructure. Strict firewall rules, PAM, and deny-by-default posture required.
